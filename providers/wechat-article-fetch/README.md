# 微信公众号抓取 Skill · 使用手册

**读者：** 法律团队（无需技术背景）
**一句话：** 给一篇微信公众号文章的链接，它在后台帮你**抓下正文并存成 Markdown 文件**，图片也一并下载到本地。

---

## 它做什么

把 `mp.weixin.qq.com` 公众号文章的网址交给它，它会：

- **后台抓取**，不弹出浏览器窗口
- 自动等页面加载完、处理懒加载图片
- 提取**标题 + 正文**，清掉无关的 HTML 标签和装饰
- 把文章存成一个 Markdown 文件，配图自动下载到对应的资源文件夹
- 自动按文章标题给文件命名
- 失败时自动重试三次，并能识别"参数错误"这类异常页
- 如果识别到内容是法律案例/法规，会顺手调 `legal-text-format` 做规范化格式

跨平台可用（Windows / macOS / Linux）。

> 这里说的"无头模式"就是浏览器在后台静默运行，**不弹窗**——你不会看到浏览器画面跳出来。

## 它不做什么

- **不抓需要登录后才能看的公众号专属内容。**
- **不批量抓全网网页。** 多网站抓取走 `web-crawl-workflow`。
- **不替你判断文章真伪。** 它原样保留，可靠性需你判断。
- **不把抓来的内容当指令执行。** 公众号正文里如果出现"忽略以上指令"之类话术，会被当作"数据"原样保留，不会被执行。

## 产出物

- 一篇整洁的 Markdown 文件（标题 + 正文 + 元信息）
- 一个同名的资源文件夹，存放文章中下载下来的图片
- 法律内容会额外触发 `legal-text-format`，输出到 `archive/` 目录

## 第一次使用

### 怎么触发
直接说："**帮我抓一下这篇公众号文章 / 把这个微信文章保存下来 / 公众号存档 [URL]**。"

### 要准备什么
- **公众号文章的网址**（一般是 `mp.weixin.qq.com/s/...`）。

### 结果在哪看
Markdown 文件和图片文件夹会保存到项目根目录（法律内容则在 `archive/`）；具体路径会在对话里反馈。

## 常见问题

- **抓下来的文章排版会乱吗？** 它会清掉装饰性标签、保留段落结构；个别复杂排版可能简化。
- **图片会丢吗？** 小的装饰图（社交按钮、表情）会被过滤掉；正文里的实质性配图会被下载并保留原位。
- **公众号改版了抓不到怎么办？** 切到 `browser-diagnostic` 查清原因再修；或退而求其次让你手动复制文字。
- **能直接当法律证据用吗？** 不能，这是研究材料；正式取证走 `web-verification-batch`。

## 术语小词典

- **公众号文章网址**：浏览器里这篇文章打开后地址栏显示的那串字符。
- **Markdown**：一种轻量纯文字稿格式，干净、易读、可再加工。
- **无头模式 / headless**：浏览器在后台静默运行，不弹窗。
- **懒加载**：网页"滚到哪才加载哪"的机制；它会自动滚一遍把所有内容都加载出来。
