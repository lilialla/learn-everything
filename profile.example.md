# Learner Profile (example)

> Copy this file to `profile.md` (which is gitignored) and fill it in with your own
> preferences. The host model reads `profile.md` to tailor how it teaches and reviews
> across your tracks. Keep anything sensitive out of the committed example.

## Background

A sentence or two about where you're starting from — prior knowledge, current role,
relevant experience. Example: "Practicing lawyer; comfortable with legal reasoning, new
to LLM/agent engineering."

## Goals

What you're trying to achieve overall (not per-track — tracks carry their own goals).
Example: "Build working intuition for agent architectures; pass IELTS at band 7.5."

## Preferred pedagogy

Your default teaching style when a track doesn't specify one.
Options: `socratic` (led by questions), `feynman` (you explain back, model probes gaps),
`active-recall` (plain recall quizzing).
Example: "Default socratic for new material, active-recall for review."

## Languages

Languages you want to learn and be quizzed in.
Example: "Explanations in Chinese, technical terms kept in English."

## Notes

Anything else the model should know — time available per day, when you focus best,
formats you dislike, etc.
